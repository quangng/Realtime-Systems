This function resets and initializes the LCD screen.

Remarks:
Timing: 14 bytes are sent over SPI at 48 MHz. The function waits until all bytes have been sent before returning.

This function reads the raw values from the touch screen controller ADC.

The returned values represent the x and y positions of the pen on the screen, and the touch pressure. These values must be calibrated to be meaningful.

Parameters:
xPointer to a variable to hold the x value on return. The written value will be between 0 and 4095.
yPointer to a variable to hold the y value on return. The written value will be between 0 and 4095.
z1Pointer to a variable to hold the first pressure value on return. The written value will be between 0 and 4095.
z2Pointer to a variable to hold the second pressure value on return. The written value will be between 0 and 4095.
Remarks:
Timing: 9 bytes are sent over SPI at 1.5 MHz. The function waits until all bytes have been sent before returning.
Theoretical: t = 9*8/1.5 = 48 us

Send bitmap data to update a display row.

Can be called multiple times to send data in several batches. StartDisplayRowUpdate_DOGM128() must have been called before using this function.

Parameters:
bytesA pointer to the bitmap data to be sent.
numThe number of bytes to send. A row consists of 128 bytes, and any further bytes will be ignored.
Remarks:
Timing: Num bytes are sent over SPI at 24 MHz. The function waits until all bytes have been sent before returning.
Theoretical: t = num*8/24 = num*333ns
Measured, per byte: t = num*870ns

This function starts a (possibly partial) update of one row of the display.

The display consists of 8 rows, each 8 pixels high. Each byte of bitmap data contains eight pixels arranged vertically.

Parameters:
rowThe number of the row to update, from 0 to 7.
colThe number of the column to start the update at, from 0
Remarks:
Timing: 3 bytes are sent over SPI at 24 MHz. The function waits until all bytes have been sent before returning.
Theoretical: t = 3*8/24 = 1 us
Measured, per byte: t = 3*870ns = 2.6 us

void UpdateDisplay_DOGM128 ( uint8_t *  bytes)

This function updates the whole display.

The display consists of 8 rows, each 8 pixels high. Each byte of bitmap data contains eight pixels arranged vertically, and the bytes are arranged in rows from top to bottom.

Parameters:
bytesA pointer to the pixel data for the display. The display consists of a total of 1024 bytes, or 8 rows of 128 bytes each.
Remarks:
Timing: (3+128)*8 bytes are sent over SPI at 24 MHz. The function waits until all bytes have been sent before returning.
Theoretical: t = (3+128)*8*8/24 = 350 us
Measured, per byte: t = (3+128)*8*870 ns = 912 us
Measured, total: t = ~1000 us
